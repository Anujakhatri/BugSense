## 📌 What is this project?
An AI-powered platform that predicts where bugs are likely to occur in software before release, classifies defects by severity and type, and monitors real-time release quality using historical data and code metrics. It acts as an intelligent early-warning system for software engineering teams. 

**Bonus Feature:** It also has a **"Roast" mode**. When you submit code, you get the same serious ML-backed analysis wrapped in sarcastic AI commentary.

### 🛠 Tech Stack
- **Backend:** Python, Django REST Framework
- **Database:** PostgreSQL
- **Frontend:** React
- **Infrastructure:** Docker
- **AI/ML:** Groq API (Llama 3), XGBoost / Random Forest
- **Auth:** JWT + RBAC, django-allauth (GitHub OAuth + Google OAuth)

---

## ⚙️ How It Works
bugchetana takes two main inputs:
1. **A bug report or code snippet:** When submitted, the ML model classifies it by **Severity** (critical/high/medium/low) and **Bug Type** (authentication failure, runtime error, etc.).
2. **Historical release data:** The system tracks bugs linked to a version and computes a quality score. If a release has too many high-severity bugs, it gets flagged.

### 🔥 Why the Roast Layer?
Serious bug reports get skimmed. A report saying *"NullPointerException at line 47, severity: high"* is easily ignored. But one that says:
> *"Line 47 is crying in the corner. You passed None into a function that clearly expected a string. This is the third time this week."*
gets read.

The roast is generated by Llama 3 via Groq, using the ML output as input. The model already knows what's wrong—it just explains it in a way that sticks.

## 🏗 System Architecture

```text
User submits code or bug report
        │
        ▼
Django REST API receives request
        │
        ▼
ML Model (XGBoost / Random Forest)
  - Classifies severity & bug type
  - Computes module risk score
        │
        ▼
Groq API (Llama 3) — Roast Mode only
  - Receives ML output as structured input
  - Returns sarcastic commentary grounded in actual findings
        │
        ▼
Results saved to PostgreSQL
        │
        ▼
React Dashboard
  - Bug list with severity + type
  - Release quality score & trend chart
  - Roast text display
```

---

## 💻 React Dashboard
- **Release Overview:** Quality score (0–100), bug count by severity, and Go / No-Go indicator.
- **Quality Trends:** Line chart showing the quality score across the last N releases.
- **Module Risk Map:** Table of modules ranked by risk score (tells QA where to focus).
- **Bug Detail View:** Shows predicted severity, type, fix suggestion, and an optional collapsible Roast text.

## 🔐 Authentication & RBAC

bugchetana supports **Email + Password**, **GitHub OAuth**, and **Google OAuth** (all returning a JWT).

By default, every new user is assigned the `developer` role.

### Roles & Permissions
- **Developer:** Submit bug reports, view own submissions, use Roast Mode.
- **QA:** View all bug reports for a release, see module risk table.
- **Release Manager:** View quality dashboard, set go/no-go threshold, upgrade other users' roles.
